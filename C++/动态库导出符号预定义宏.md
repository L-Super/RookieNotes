在编写跨平台的 C++ 动态库时（尤其是 Windows 平台），你需要告诉编译器哪些函数/类是要“导出”（给别人用的），哪些是“导入”（用别人的）。通常会在 C++ 头文件中写类似这样的代码：
```cpp
#if defined(_WIN32)
    #if defined(NP_TUNNEL_EXPORTS) 
        // 如果定义了 NP_TUNNEL_EXPORTS，说明正在编译库本身，需要导出符号
        #define NP_TUNNEL_API __declspec(dllexport)
    #else
        // 如果没有定义，说明是外部程序在包含此头文件，需要导入符号
        #define NP_TUNNEL_API __declspec(dllimport)
    #endif
#else
    // Linux/macOS 下的默认可见性设置
    #define NP_TUNNEL_API __attribute__((visibility("default")))
#endif

// 使用这个宏来修饰你的函数或类
NP_TUNNEL_API void start_tunnel();
```

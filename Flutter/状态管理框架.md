## Provider
对 [InheritedWidget](https://api.flutter-io.cn/flutter/widgets/InheritedWidget-class.html) 组件的上层封装，使其更易用，更易复用。

使用 `provider` 而非手动书写 [InheritedWidget](https://api.flutter-io.cn/flutter/widgets/InheritedWidget-class.html)，有以下的优势:

- 简化的资源分配与处置
- 懒加载
- 创建新类时减少大量的模板代码
- 支持 DevTools
- 更通用的调用 [InheritedWidget](https://api.flutter-io.cn/flutter/widgets/InheritedWidget-class.html) 的方式（参考 [Provider.of](https://pub.flutter-io.cn/documentation/provider/latest/provider/Provider/of.html)/[Consumer](https://pub.flutter-io.cn/documentation/provider/latest/provider/Consumer-class.html)/[Selector](https://pub.flutter-io.cn/documentation/provider/latest/provider/Selector-class.html)）
- 提升类的可扩展性，整体的监听架构时间复杂度以指数级增长（如 [ChangeNotifier](https://api.flutter-io.cn/flutter/foundation/ChangeNotifier-class.html)，其复杂度为 O (N)）
## GetX

 GetX 是 Flutter 上的一个轻量且强大的解决方案：高性能的状态管理、智能的依赖注入和便捷的路由管理。

 GetX 有 3 个基本原则：
- **性能：** GetX 专注于性能和最小资源消耗。GetX 打包后的 apk 占用大小和运行时的内存占用与其他状态管理插件不相上下。
- **效率：** GetX 的语法非常简捷，并保持了极高的性能，能极大缩短你的开发时长。
- **结构：** GetX 可以将界面、逻辑、依赖和路由完全解耦，用起来更清爽，逻辑更清晰，代码更容易维护。
### 核心特性

| 模块         | 功能                           | 特点                       |
| ------------ | ------------------------------ | -------------------------- |
| **状态管理** | 响应式/简单状态管理            | 自动更新 UI，零代码侵入    |
| **路由管理** | 页面跳转、命名路由、中间件     | 无需 Context，支持双向绑定 |
| **依赖注入** | 单例/懒加载/生命周期管理       | 自动释放资源               |
| **实用工具** | 国际化、主题、Snackbar、Dialog | API 简洁，功能丰富         |

### 状态管理三种方式对比

#### 简单状态管理（GetBuilder）

```dart
// Controller
class CounterController extends GetxController {
  int count = 0;
  
  void increment() {
    count++;
    update();  // 手动触发更新
  }
}

// View
class CounterPage extends StatelessWidget {
  final controller = Get.put(CounterController());
  
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      body: GetBuilder<CounterController>(
        builder: (ctrl) => Text('${ctrl.count}'),
      ),
      floatingActionButton: FloatingActionButton(
        onPressed: controller.increment,
      ),
    );
  }
}
```

#### 响应式状态管理（Obx）

```dart
// Controller
class CounterController extends GetxController {
  // ".obs"(ervables)变量称为 Rx
  var count = 0.obs;  // .obs 包装为响应式
  
  void increment() => count++;  // 自动触发更新
}

// View
class CounterPage extends StatelessWidget {
  final controller = Get.put(CounterController());
  
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      // Obx()(Rx 的 "观察者")
      body: Obx(() => Text('${controller.count}')),
      floatingActionButton: FloatingActionButton(
        onPressed: controller.increment,
      ),
    );
  }
}
```
只有当 _Rx_ 变量的值发生变化时，**GetX**才会更新界面。
#### 混合使用

```dart
class HybridController extends GetxController {
  // 响应式变量（自动追踪）
  var reactiveCount = 0.obs;
  
  // 普通变量（手动更新）
  int manualCount = 0;
  
  void updateReactive() => reactiveCount++;
  
  void updateManual() {
    manualCount++;
    update();  // GetBuilder 需要
  }
}
```

### 路由管理

#### 基础导航

```dart
// 跳转页面
Get.to(NextPage());

// 替换页面
Get.off(NextPage());

// 清空栈并跳转
Get.offAll(HomePage());

// 返回上一页
Get.back();

// 带参数跳转
Get.to(DetailPage(), arguments: {'id': 123});
// 接收参数
var args = Get.arguments;  // {'id': 123}

// 或
var data = await Get.to(Payment());
```

#### 命名路由

```dart
// 配置路由表
void main() {
  // 要定义路由，使用GetMaterialApp
  runApp(GetMaterialApp(
  	// 定义路由
    initialRoute: '/',
    getPages: [
      GetPage(name: '/', page: () => HomePage()),
      GetPage(name: '/detail', page: () => DetailPage()),
    ],
  ));
}

// 跳转
Get.toNamed('/detail', arguments: {'id': 123});
// 在类或 controler 获取
print(Get.arguments); // {'id': 123}

// 动态路由
GetPage(
  name: '/user?device=phone&id=123',
  page: () => UserPage(),
)
print(Get.parameters['id']); // 123

GetPage(
  name: '/user/:id',
  page: () => UserPage(),
)
// 访问 /user/123，
Get.parameters['id'] = '123'
```

#### 路由中间件
想通过监听 Get 事件来触发动作，你可以使用 routingCallback 来实现。如果没有使用 GetMaterialApp，你可以使用手动 API 来附加 Middleware 观察器。

```dart
class AuthMiddleware extends GetMiddleware {
  @override
  RouteSettings? redirect(String? route) {
    final isLogged = Get.find<AuthController>().isLogged.value;
    return isLogged ? null : RouteSettings(name: '/login');
  }
}

GetPage(
  name: '/profile',
  page: () => ProfilePage(),
  middlewares: [AuthMiddleware()],
)
```

### 依赖注入

#### 注册方式
```dart
// 方式一：立即注册
Get.put(MyController());

// 方式二：懒加载（首次使用时创建）
Get.lazyPut(() => MyController());

// 方式三：单例模式
Get.put(MyController(), permanent: true);

// 方式四：异步注册
Get.putAsync(() async => await MyController.create());

// 方式五：工厂模式（每次获取新实例）
Get.create(() => MyController());
```
##### `Get.put`
put 可设置的选项：
```dart
Get.put<S>(
  // 想得到保存的类，比如控制器或其他东西。
  // 注："S "意味着它可以是任何类型的类。
  S dependency

  // 可选：当你想要多个相同类型的类时，可以用这个方法。
  // 因为你通常使用Get.find<Controller>()来获取一个类。
  // 你需要使用标签来告诉你需要哪个实例。
  // 必须是唯一的字符串
  String tag,

  // 可选：默认情况下，get会在实例不再使用后进行销毁
  // （例如：一个已经销毁的视图的Controller)
  // 但你可能需要这个实例在整个应用生命周期中保留在那里，就像一个sharedPreferences的实例或其他东西。
  //所以你设置这个选项
  // 默认值为false
  bool permanent = false,

  // 可选：允许你在测试中使用一个抽象类后，用另一个抽象类代替它，然后再进行测试。
  // 默认为false
  bool overrideAbstract = false,

  // 可选：允许你使用函数而不是依赖（dependency）本身来创建依赖。
  // 这个不常用
  InstanceBuilderCallback<S> builder,
)
```
##### `Get.putAsync`
```dart
Get.putAsync<S>(
  // 一个将被执行的异步方法，用于实例化你的类。
  AsyncInstanceBuilderCallback<S> builder,

  // 可选：和Get.put()一样，当你想让同一个类有多个不同的实例时，就会用到它。
  // 必须是唯一的
  String tag,

  // 可选：与Get.put()相同，当你需要在整个应用程序中保持该实例的生命时使用。
  // 默认值为false
  bool permanent = false
)
```
##### `Get.Create`
```dart
Get.create<S>(
  // 一个返回每次调用"Get.find() "都会被新建的类的函数。
  // 示例: Get.create<YourClass>(()=>YourClass())
  FcBuilderFunc<S> builder,

  // 可选：就像Get.put()一样，但当你需要多个同类的实例时，会用到它。
  // 当你有一个列表，每个项目都需要自己的控制器时，这很有用。
  // 需要是一个唯一的字符串。只要把标签改成名字
  String name,

  // 可选：就像 Get.put() 一样，
  // 它是为了当你需要在整个应用中保活实例的时候
  // 区别在于 Get.create 的 permanent默认为true
  bool permanent = true
```

### 高级特性

#### Worker（监听变化）

```dart
class MyController extends GetxController {
  var count = 0.obs;
  
  @override
  void onInit() {
    super.onInit();
    
    // 值变化时触发
    ever(count, (value) => print('count: $value'));
    
    // 首次变化触发
    once(count, (value) => print('首次变化: $value'));
    
    // 防抖（变化停止后 1 秒触发）
    debounce(count, (value) => print('停止变化'), time: Duration(seconds: 1));
    
    // 节流（1 秒内最多触发一次）
    interval(count, (value) => print('节流'), time: Duration(seconds: 1));
  }
}
```

#### 全局状态绑定

```dart
// 无需 BuildContext 即可访问状态
class GlobalService extends GetxService {
  var user = User().obs;
  
  Future<void> login(String name) async {
    user.update((u) => u?.name = name);
  }
}

// 任何地方都可以访问
Get.find<GlobalService>().user.value.name;
```

### 常见问题

| 问题              | 解决方案                                           |
| ----------------- | -------------------------------------------------- |
| UI 不更新         | 使用 `update()` 或确保 `.obs` 变量带 `.value`      |
| 内存泄漏          | 使用 `Get.put` 替代手动创建，让 GetX 管理生命周期  |
| 路由传参丢失      | 使用 `Get.toNamed` + `arguments`，或动态路由 `:id` |
| GetBuilder 不更新 | 检查是否调用了 `update()`，Controller 是否正确注册 |

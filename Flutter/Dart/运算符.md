
## 类型测试运算符

`as`、`is` 和 `is!` 运算符在运行时检查类型时非常有用。

|运算符|含义|
|---|---|
| `as` |类型转换（也用于指定 [library prefix(库前缀)](https://dart.ac.cn/language/libraries#specifying-a-library-prefix)）|
| `is` |如果对象具有指定 type，则为 true|
| `is!` |如果对象不具有指定类型，则为 true|

如果 `obj` 实现了 `T` 指定的接口，则 `obj is T` 的结果为 true。例如，`obj is Object?` 始终为 **true**。

仅当你确定对象是该类型时，才使用 `as` 运算符将对象强制转换为特定 type。示例

```dart
(employee as Person).firstName = 'Bob';
// `employee` 为 null 或不是 `Person` 类型，则会抛出异常
```


如果你不确定对象是否为 `T` 类型，请在使用对象之前使用 `is T` 检查类型。

```dart
if (employee is Person) {
  // Type check
  employee.firstName = 'Bob';
}
```
## 条件表达式

Dart 有两个运算符，可以简洁地评估表达式

-  `condition ? expr1 : expr2` 
- `expr1 ?? expr2` ： if-null 运算符 `??`（也称为空值合并运算符）。如果 expr1 不为 null，则返回其值；否则，评估并返回 expr2 的值。

当你需要根据布尔表达式赋值时，请考虑使用条件运算符 `?` 和 `:`。

```dart
var visibility = isPublic ? 'public' : 'private';

// name 是否为 null
String playerName(String? name) => name ?? 'Guest';
// 等效于
String playerName(String? name) => name != null ? name : 'Guest';
```

## 级联表示法

级联 (`..`, `?..`) 允许你对同一个对象进行一系列操作。除了访问实例成员，你还可以在同一个对象上调用实例方法。这通常可以省去创建临时变量的步骤，并允许你编写更流畅的代码。

示例：

```dart
// 构造函数 `Paint()` 返回一个 `Paint` 对象。
// 级联表示法后面的代码对该对象进行操作，忽略任何可能返回的值
var paint = Paint()
  ..color = Colors.black
  ..strokeCap = StrokeCap.round
  ..strokeWidth = 5.0;
  
// 等效于
var paint = Paint();
paint.color = Colors.black;
paint.strokeCap = StrokeCap.round;
paint.strokeWidth = 5.0;

var list = <int>[1, 2, 3]
    ..add(4)
    ..addAll([5, 6])
    ..removeAt(0);
```

如果级联操作的对象可能为 null，则在第一个操作中使用空短路级联 (`?..`)。以 `?..` 开头可确保不会对该 null 对象尝试任何级联操作。

```dart
class Person {
  String? name;
  int? age;
  
  void printInfo() {
    print('$name, $age');
  }
}

Person? getPerson(bool returnNull) {
  return returnNull ? null : Person();
}

void main() {
  var person = getPerson(true);
 
  // 使用 ?.. 安全级联
  person
    ?..name = 'Tom'
    ..age = 20;  // 注意：后面的 .. 会自动继承 ?.. 的效果
  
  person?.printInfo(); // 不会执行，因为 person 是 null
  
  // 非空情况
  var person2 = getPerson(false);
  person2
    ?..name = 'Jerry'
    ..age = 25;
  person2?.printInfo(); // Jerry, 25
}
```

还可以嵌套 cascade (级联)。例如：

```dart
final addressBook =
    (AddressBookBuilder()
          ..name = 'jenny'
          ..email = 'jenny@example.com'
          ..phone =
              (PhoneNumberBuilder()
                    ..number = '415-555-0100'
                    ..label = 'home')
                  .build())
        .build();
```

请注意将级联构建在返回实际 object 的函数上。例如，以下代码将失败

```
var sb = StringBuffer();
sb.write('foo')
  ..write('bar'); // Error: `write()` 调用返回 void，不能在 `void` 上构建级联。
```

> [!note] 
> 严格来说，级联的“双点”表示法并不是运算符。它只是 Dart 语法的一部分。

## 展开运算符

展开运算符评估一个产生集合的表达式，解包结果值，并将其插入到另一个集合中。

**展开运算符实际上不是运算符表达式**。`...` / `...?` 语法是集合字面量本身的一部分。因此，你可以在 [collection(集合)](https://dart.ac.cn/language/collections#spread-operators) 了解更多信息。

因为它不是运算符，所以该语法没有任何“运算符优先级”。实际上，它具有最低的“优先级”——任何类型的表达式都可以作为展开目标，例如

```
[...a + b]
```

## 其他运算符

| 运算符   | 名称      | 含义                                                                                                                     |
| ----- | ------- | ---------------------------------------------------------------------------------------------------------------------- |
| `?[]` | 条件下标访问  | 类似于 `[]`，但最左边的操作数可以为 null；示例：`fooList?[1]` 将整数 `1` 传递给 `fooList` 以访问索引为 `1` 的元素，除非 `fooList` 为 null（在这种情况下表达式评估为 null） |
| `?.`  | 条件成员访问  | 类似于 `.`，但最左边的操作数可以为 null；示例：`foo?.bar` 从表达式 `foo` 中选择属性 `bar`，除非 `foo` 为 null（在这种情况下 `foo?.bar` 的值为 null）              |
| `!`   | 非空断言运算符 | 将表达式转换为其底层非空类型，如果转换失败则抛出运行时异常；示例：`foo!.bar` 断言 `foo` 非空并选择属性 `bar`，除非 `foo` 为 null（在这种情况下会抛出运行时异常）                     |

有关 `.`、`?.` 和 `..` 运算符的更多信息，请参阅 [class](https://dart.ac.cn/language/classes)。

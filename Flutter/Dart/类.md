Dart 是一种面向对象的语言，支持类和基于混入（mixin）的继承。[extension](https://dart.ac.cn/language/extension-methods)方法是一种在不改变类或创建子类的情况下向类添加功能的方式。[类修饰符](https://dart.ac.cn/language/class-modifiers) 允许你控制库如何子类型化一个类。 

## 基本形式

```dart
class Point {
  double x = 0;
  double? y; // Declare y, initially null.
  double z = 0; // Declare z, initially 0.
}

var p = Point(3, 4);
assert(p.y == 2);

// 使用 ?. 而不是 . 来避免左侧操作数为 null 时发生异常
// If p is non-null, set a variable equal to its y value.
var a = p?.y;
```

> [!tip]
>
> 在运行时获取对象的类型，你可以使用 `Object` 属性 `runtimeType`，它会返回一个 [`Type`](https://api.dart.ac.cn/dart-core/Type-class.html) 对象。在生产环境中，推荐使用[类型测试运算符](https://dart.ac.cn/language/operators#type-test-operators)来测试对象的类型。即 `object is Type` 比 `object.runtimeType == Type` 更稳定。

## 构造函数

构造函数名称可以是 `ClassName` 或 `ClassName.identifier`

```dart
// 默认构造
class Point {
  double x, y;
}

// 标准构造/生成式构造
class Point {
  double x, y;
  Point(this.x, this.y);
}

// 命名构造
// 使用命名构造函数可以为一个类实现多个构造函数或提供额外的清晰度
class Point {
  double x, y;
  Point.origin() : x = 0, y = 0;
}

// 重定向构造
class Point {
  double x, y;
  Point(this.x, this.y);
  Point.alongX(double x) : this(x, 0);
}

// 常量构造
// 定义一个 const 构造函数，并将所有实例变量设置为 final
class ImmutablePoint {
  final double x, y;
  const ImmutablePoint(this.x, this.y);
}

// 工厂构造
// 使用 factory 关键字
class Logger {
  static final Map<String, Logger> _cache = {};
  final String name;
  factory Logger(String name) {
    return _cache.putIfAbsent(name, () => Logger._internal(name));
  }
  Logger._internal(this.name);
}
```

默认构造函数是没有参数或名称的生成式构造函数。

子类不继承父类的命名构造函数。要在子类中创建父类中定义的命名构造函数，请在子类中实现该构造函数。

工厂构造函数无法访问 `this`。

## 实例变量

使用可空类型声明的未初始化实例变量的值为 `null`。非空实例变量必须在声明时或构造函数中初始化。

在声明中初始化实例变量：

```dart
class PointA {
  double x = 1.0;
  double y = 2.0;
}
```

### 实例变量初始化   

#### 使用初始化形式参数。
在构造函数声明中，包含 `this.<propertyName>` 并省略函数体。`this` 关键字指代当前实例。
当存在名称冲突时，使用 `this`。否则，Dart 风格省略 `this`。生成式构造函数有一个例外，你必须以 `this` 为前缀初始化形式参数名称。
初始化形式参数还允许你初始化不可为空或 `final` 的实例变量。这两种类型的变量都需要初始化或默认值。
```dart
class PointB {
  final double x;
  final double y;

  // Sets the x and y instance variables
  // before the constructor body runs.
  PointB(this.x, this.y);

  // Initializing formal parameters can also be optional.
  PointB.optional([this.x = 0.0, this.y = 0.0]);
}
```
通过初始化形式参数引入的所有变量都是 final 的，并且仅在其初始化变量的作用域内。
#### 使用初始化列表
```dart
// Initializer list sets instance variables before
// the constructor body runs.
Point.fromJson(Map<String, double> json) : x = json['x']!, y = json['y']! {
  print('In Point.fromJson(): ($x, $y)');
}
```

---

在声明处初始化非 `late` 实例变量会在实例创建时设置其值，这发生在构造函数及其初始化列表执行之前。因此，非 `late` 实例变量的初始化表达式（在 `=` 之后）不能访问 `this`。

```dart
double initialX = 1.5;

class Point {
  // OK, can access declarations that do not depend on `this`:
  double? x = initialX;

  // ERROR, can't access `this` in non-`late` initializer:
  double? y = this.x;

  // OK, can access `this` in `late` initializer:
  late double? z = this.x;

  // OK, `this.x` and `this.y` are parameter declarations, not expressions:
  Point(this.x, this.y);
}
```

实例变量可以是 `final` 类型，在这种情况下它们必须且只能被设置一次。初始化 `final`、非 `late` 实例变量可以在声明时进行，也可以通过构造函数参数或使用构造函数的[初始化列表](https://dart.ac.cn/language/constructors#use-an-initializer-list)进行。

静态变量在被使用之前不会被初始化。

## 方法

```dart
class Calculator {
  // 实例方法
  int add(int a, int b) => a + b;
  
  // 静态方法
  static int subtract(int a, int b) => a - b;
  
  // getter/setter
  String get name => 'Calculator';
  set value(int v) => _value = v;
}
```

静态方法（类方法）不作用于实例，因此无法访问 `this`

### Getter 与 Setter

所有实例变量都会生成一个隐式的 `getter` 和 `setter` 方法。除了 final 实例变量和没有初始化器的 `late final` 实例变量。

```dart
class Rectangle {
  double left, top, width, height;

  Rectangle(this.left, this.top, this.width, this.height);

  // Define two calculated properties: right and bottom.
  double get right => left + width;
  set right(double value) => left = value - width;
  double get bottom => top + height;
  set bottom(double value) => top = value - height;
}

void main() {
  var rect = Rectangle(3, 4, 20, 15);
  assert(rect.left == 3);
  rect.right = 12;
  assert(rect.left == -8);
}
```

### 操作符重载
声明一个运算符，先使用内置标识符 `operator`，然后是要定义的运算符。
```dart
class Vector {
  final int x, y;
  Vector(this.x, this.y);
  
  Vector operator +(Vector other) => Vector(x + other.x, y + other.y);
  Vector operator -(Vector other) => Vector(x - other.x, y - other.y);
}
```

## 继承

```dart
class Animal {
  void eat() => print('Eating');
}

class Dog extends Animal {
  @override
  void eat() => print('Dog eating');
  
  void bark() => print('Barking');
}
```

## 抽象类与接口
实例、Getter 和 Setter 方法可以是抽象的

```dart
// 抽象类（不能实例化）
abstract class Shape {
  double get area;
  void draw();  // 抽象方法
}

// 隐式接口（所有类都是接口）
class Circle implements Shape {
  @override
  double get area => 3.14 * radius * radius;
  
  @override
  void draw() => print('Drawing circle');
}

// 指定一个类实现多个接口
class Point implements Comparable, Location {
  ...
}
```

## Mixin
定义混入，使用 `mixin` 声明。

使用混入时，使用 `with` 关键字，后跟一个或多个混入名称。
```dart
mixin Swimmer {
  void swim() => print('Swimming');
}

mixin Flyer {
  void fly() => print('Flying');
}

class Duck extends Animal with Swimmer, Flyer {}
```

## 访问修饰符
```dart
class MyClass {
  int public = 1;           // 公开
  int _private = 2;         // 库私有（以下划线开头）
  static int staticVar = 3; // 静态变量
  final int constant = 4;   // 只能赋值一次
}
```

## 扩展方法
使用 `extends` 创建子类，并使用 `super` 引用父类
```dart
extension NumberParsing on String {
  int toInt() => int.parse(this);
}
```
子类可以重写实例方法（包括运算符）、getter 和 setter。

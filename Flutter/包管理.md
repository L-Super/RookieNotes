包管理是 Dart 和 Flutter 生态系统的核心，允许开发者复用代码、管理依赖、共享库。

## 核心概念

### 什么是包？

包是一组 Dart 文件的集合，提供特定功能。包分为：

| 类型       | 说明                      | 示例                           |
| ---------- | ------------------------- | ------------------------------ |
| **应用包** | 可执行的应用程序          | 你的 Flutter 项目              |
| **库包**   | 供其他项目使用的库        | `http`, `provider`             |
| **插件包** | 包含原生代码的 Flutter 包 | `camera`, `shared_preferences` |

### 包的来源

- **pub.dev**：官方包仓库（类似 npm）
- **Git 仓库**：GitHub、GitLab 等
- **本地路径**：文件系统中的包
- **私有仓库**：企业内部包

## 核心配置文件

### `pubspec.yaml` - 包配置文件

```yaml
name: my_app                    # 包名（必填）
description: A new Flutter project  # 描述
version: 1.0.0+1               # 版本号（应用包可选）

environment:                    # SDK 版本约束（必填）
  sdk: '>=3.0.0 <4.0.0'
  flutter: '>=3.0.0'

dependencies:                   # 生产环境依赖
  flutter:
    sdk: flutter
  http: ^1.1.0                  # ^ 表示兼容版本
  provider: ^6.0.5
  shared_preferences: ^2.2.0

dev_dependencies:               # 开发环境依赖
  flutter_test:
    sdk: flutter
  flutter_lints: ^3.0.0

dependency_overrides:           # 强制版本覆盖
  http: 1.0.0

flutter:                        # Flutter 特定配置
  uses-material-design: true
  assets:
    - assets/images/
    - assets/fonts/
```

## 依赖管理

### 添加依赖

```yaml
dependencies:
  # 1. pub.dev 上的包（最常用）
  http: ^1.1.0                 # 兼容最新版本
  intl: 0.18.0                 # 固定版本
  path: ^1.9.0                
  
  # 2. Git 仓库
  flutter_staggered_grid_view:
    git:
      url: https://github.com/letsar/flutter_staggered_grid_view.git
      ref: main               # 分支/tag/commit
  
  # 3. 本地路径
  my_local_package:
    path: ../my_package
  
  # 4. SDK 内置包
  flutter:
    sdk: flutter
  
  # 5. 禁用版本约束
  device_info_plus: any       # 使用最新版本（不推荐）
```

>  Pub（https://pub.dev/ ）是 Google 官方的 Dart Packages 仓库
### 版本约束语法

| 语法             | 含义                     | 示例版本匹配       |
| ---------------- | ------------------------ | ------------------ |
| `^1.2.3`         | 兼容版本（主版本号不变） | 1.2.3 ~ 2.0.0 之前 |
| `>=1.2.3 <2.0.0` | 范围约束                 | 1.2.3, 1.5.0       |
| `1.2.3`          | 固定版本                 | 只有 1.2.3         |
| `any`            | 任意版本                 | 最新可用版本       |
| `>1.2.3`         | 大于指定版本             | 1.2.4+             |

## 常用命令

### 获取依赖

```bash
# 获取所有依赖（首次运行）
flutter pub get

# 或（Dart 项目）
dart pub get
```

### 更新依赖

```bash
# 更新到兼容的最新版本
flutter pub upgrade

# 更新到最新版本（可能不兼容）
flutter pub upgrade --major-versions

# 列出过时的依赖
flutter pub outdated
```

### 添加/移除依赖

```bash
# 添加依赖（自动修改 pubspec.yaml）
flutter pub add http
flutter pub add provider --dev      # 添加到 dev_dependencies

# 移除依赖
flutter pub remove http

# 添加 Git 依赖
flutter pub add my_package --git-url https://github.com/user/my_package.git
```

### 其他命令

```bash
# 查看已安装的依赖树
flutter pub deps

# 缓存管理
flutter pub cache repair        # 修复缓存
flutter pub cache clean         # 清理缓存（旧版本）

# 全局运行工具
dart pub global activate webdev
```



## 依赖解析过程

### 依赖获取流程

```mermaid
graph LR
    A[pub get] --> B[读取 pubspec.yaml]
    B --> C[解析版本约束]
    C --> D[获取依赖的依赖]
    D --> E[解决版本冲突]
    E --> F[下载包]
    F --> G[生成 pubspec.lock]
```

### `pubspec.lock` 文件

锁定精确版本，确保团队环境一致：

```yaml
# 示例
packages:
  http:
    version: 1.1.0
    source: hosted
    description:
      url: "https://pub.dev"
  path:
    version: 1.9.0
    source: hosted
```

**重要：** `pubspec.lock` 应该提交到版本控制（应用包），库包通常忽略。



## 包的存放位置

### 全局缓存

Dart 包存储在用户缓存目录：

- **Windows**: `%LOCALAPPDATA%\Pub\Cache`
- **macOS/Linux**: `~/.pub-cache`

```bash
# 查看缓存位置
dart pub cache list

# 手动清理缓存
rm -rf ~/.pub-cache  # 谨慎操作，会删除所有包
```

### 项目本地

`.dart_tool/package_config.json` 记录包的解析路径。

## 版本冲突解决

### 常见冲突场景

```yaml
# 主项目依赖
dependencies:
  package_a: ^1.0.0   # 依赖 package_b: ^2.0.0
  package_c: ^2.0.0   # 依赖 package_b: ^1.0.0
```

如果 `package_a` 和 `package_c` 依赖不同版本的 `package_b`，Pub 会尝试找到**共享兼容版本**。

### 解决方案

1. **使用 `dependency_overrides`**（强制覆盖）

```yaml
dependency_overrides:
  package_b: 2.0.0   # 强制使用这个版本
```

2. **升级/降级依赖**
```bash
flutter pub upgrade package_a
```

3. **使用 `^` 语法扩大版本范围**

## 创建自己的包

### 创建库包

```bash
flutter create --template=package my_package
# 或
dart create -t package my_package
```

### 发布到 pub.dev

```bash
# 检查是否准备好
flutter pub publish --dry-run

# 发布
flutter pub publish
```

### 关键文件结构

```

├── test/                  # 单元测试
├── example/               # 示例应用
├── pubspec.yaml
├── README.md
└── CHANGELOG.md
```

## 常见问题

### Q1: `pub get` 很慢怎么办？

```bash
# 配置国内镜像（macOS/Linux）
export PUB_HOSTED_URL=https://pub.flutter-io.cn
flutter pub get

# 或使用 Flutter 官方中国镜像
flutter pub get --pub-mirror=https://pub.flutter-io.cn
```

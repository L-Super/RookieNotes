## 什么是内核对象

内核对象（Kernel Object）是 Windows 操作系统内核管理的数据结构，用于表示系统资源，如进程、线程、文件、互斥体等。这些对象由内核创建和维护，应用程序通过句柄（Handle）来访问它们。

内核对象可以供系统和应用程序使用来管理各种各样的资源，比如进程、线程和文件等。

内核对象的数据结构只能被内核访问，因此应用程序无法在内存中找到这些数据结构并直接改变它们的内容。Windows 提供了一组函数，以便对这些结构进行操作。内核对象始终都可以通过这些函数进行访问。当调用创建内核对象的函数时，会返回一个用于标识该对象的句柄。进程中的任何线程都可以使用这个句柄。将这个句柄传递给 Windows 的各个函数，系统就能知道你想操作哪个内核对象。

### 引用计数

内核知道有多少进程正在使用某个内核对象，因为每个对象的数据结构包含一个引用计数。当一个对象刚创建时，它的引用计数被置为 1。然后，当另一个进程访问一个现有的内核对象时，计数就递增 1。当进程终止运行时，内核自动确定该进程仍然打开的所有内核对象的引用计数。如果内核对象的引用计数降为 0，内核就撤消该对象。这样可以确保在没有进程引用该对象时系统中不保留任何内核对象。

### 安全性

内核对象能够得到安全描述符的保护。安全描述符用于描述谁创建了该对象，谁能够访问或使用该对象，谁无权访问该对象。安全描述符通常在编写服务端时使用，如果编写客户端，可以忽略内核对象的这个特性。

## 核心特征

1. **由内核管理**：存在于内核地址空间，不能直接访问
2. **引用计数**：使用引用计数机制管理生命周期
3. **安全性**：具有安全描述符，支持访问控制
4. **句柄访问**：应用程序通过句柄间接操作

## 常见内核对象

用于创建内核对象的所有函数均返回与进程相关的句柄，该句柄值实际上是放入进程的句柄表中的索引，它用于标识内核对象的信息存放的位置。当调试一个应用程序并且观察内核对象句柄的实际值时，会看到一些较小的值，如 1，2 等。

```cpp
// 进程对象
HANDLE hProcess = OpenProcess(PROCESS_ALL_ACCESS, FALSE, processId);

// 线程对象
HANDLE hThread = CreateThread(nullptr, 0, ThreadProc, nullptr, 0, nullptr);

// 文件对象
HANDLE hFile = CreateFile(L"test.txt", GENERIC_READ, 0, 
                          nullptr, OPEN_EXISTING, 0, nullptr);

// 事件对象
HANDLE hEvent = CreateEvent(nullptr, TRUE, FALSE, L"MyEvent");

// 互斥体
HANDLE hMutex = CreateMutex(nullptr, FALSE, L"MyMutex");

// 信号量
HANDLE hSemaphore = CreateSemaphore(nullptr, 1, 10, L"MySemaphore");
```

### 主要内核对象类型

| 对象类型 | 用途 | 创建 API |
|---------|------|---------|
| Process | 进程管理 | CreateProcess |
| Thread | 线程管理 | CreateThread |
| File | 文件 I/O | CreateFile |
| Event | 同步通知 | CreateEvent |
| Mutex | 互斥访问 | CreateMutex |
| Semaphore | 资源计数 | CreateSemaphore |
| Job | 进程组管理 | CreateJobObject |
| Section | 内存映射 | CreateFileMapping |

## 句柄管理

```cpp
// 创建进程对象
HANDLE hProcess = OpenProcess(PROCESS_ALL_ACCESS, FALSE, processId);

// 句柄继承
SECURITY_ATTRIBUTES sa;
sa.nLength = sizeof(SECURITY_ATTRIBUTES);
sa.bInheritHandle = TRUE;  // 允许子进程继承
sa.lpSecurityDescriptor = nullptr;

HANDLE hEvent = CreateEvent(&sa, TRUE, FALSE, nullptr);

// 句柄复制
HANDLE hDuplicate;
DuplicateHandle(GetCurrentProcess(), hEvent,
                GetCurrentProcess(), &hDuplicate,
                0, FALSE, DUPLICATE_SAME_ACCESS);

// 关闭句柄
CloseHandle(hEvent);
CloseHandle(hDuplicate);
```

如果调用一个函数创建内核对象，但调用失败，返回的句柄值通常是 0 （`NULL`）。发生这种情况是因为系统的内存非常短缺，或者遇到了安全方面的问题。不过有少数函数在运行失败时返回的句柄值是-1（`INVALID_HANDLE_VALUE`）。

例如，**如果 `CreateFile()` 未能打开指定的文件，它将返回 `INVALID_HANDLE_VALUE`，而不是返回 `NULL`**。当查看创建内核对象的函数返回值时，必须格外小心。

### 关闭内核对象

无论怎样创建内核对象，都要调用 `CloseHandle()` 来结束对该对象的操作：

```cpp
BOOL CloseHandle(HANDLE hobj);
```

该函数首先检查调用进程的句柄表，以确保传递给它的索引（句柄）用于标识一个进程实际上无权访问的对象。如果该索引有效，那么系统获得内核对象的数据结构的地址，并确定该结构中的引用计数的数据成员。如果引用计数是 0，便从内存中撤消该内核对象。 

如果将一个无效句柄传递给 `CloseHanle()`，将会有两种情况。如果进程运行正常， `CloseHandle()` 返回 `FALSE`，而 `GetLastError()` 返回 `ERROR_INVALID_HANDLE`。如果进程正在排除错误，系统将通知调试程序，以便能排除它的错误。 

在 `CloseHanle()` 返回之前，它会清除进程的句柄表中的项目，该句柄对当前进程已经无效，不应该试图使用它。

假如忘记调用 `CloseHanle()` 函数，进程有可能泄漏资源（如内核对象）。但是，**当进程终止运行时，操作系统能够确保该进程使用的任何资源或全部资源均被释放**，这是有保证的。

另外，这个情况适用于所有对象、资源和内存块，也就是说，当进程终止运行时，系统将保证进程不会留下任何对象。

## 跨进程共享内核对象

许多情况下，在不同进程中运行的线程需要共享内核对象。

- 文件映射对象能够在同一台机器上运行的两个进程之间共享数据块。
- 邮箱和指定的管道使得应用程序能够在连网的不同机器上运行的进程之间发送数据块。
- 互斥对象、信标和事件使得不同进程中的线程能够同步它们的连续运行，这与一个应用程序在完成某项任务时需要将情况通知另一个应用程序的情况相同

### 对象句柄的继承性

只有当进程具有父子关系时，才能使用对象句柄的继承性。

首先，当父进程创建内核对象时，必须向系统指明，它希望对象的句柄是个可继承的句柄。 **虽然内核对象句柄具有继承性，但是内核对象本身不具备继承性。**

### 命名对象

```cpp
// 创建命名对象（可跨进程共享）
HANDLE hEvent = CreateEvent(nullptr, TRUE, FALSE, L"Global\\MyEvent");

// 打开已存在的命名对象
HANDLE hEvent2 = OpenEvent(EVENT_ALL_ACCESS, FALSE, L"Global\\MyEvent");
```

### 复制对象句柄

共享跨进程边界的内核对象的最后一个方法是使用 `DuplicateHandle()` 函数。该函数取出一个进程的句柄表中的项目，并将该项目拷贝到另一个进程的句柄表中。
